-----------------------------------------------------------------------------
 Guía de Seguridad:
-----------------------------------------------------------------------------

Este archivo es una micro guía de referencia que debería de analiar al poner
su proyecto en producción:

1) Actualice la versión de PHP a la fecha
   - Vea la revisiones de http://cve.mitre.org
2) Nunca deje un archivo con un phpinfo()
3) Nunca deje phpMyAdmin en una url pública
4) Use .htaccess para proteger directorios privados
5) Edite el archivo php.ini o utilice la función ini_set() en servidores compartidos
   - register_globals = Off
   - error_reporting = E_ALL ^ (E_STRICT | E_DEPRECATED | E_NOTICE)
   - display_errors = Off
   - log_errors = On
   - error_log = /path/privado/errors.log
   - magic_quotes_gpc = Off
   - magic_quotes_runtime = Off
   - safe_mode = Off
   - safe_mode_gid = Off
   - safe_mode_include_dir = /path/includes/permitidos/
   - safe_mode_exec_dir = /path/ejecutables/permitidos/
   - expose_php = Off
   - memory_limit = 8M
   - post_max_size = 8M
   - max_execution_time = 30
   - max_input_time = 60
   - disable_functions = show_source, exec, shell_exec, system, passthru, proc_open, popen
   - enable_dl = Off
   - file_uploads = Off (de no ser necesario)
   - max_file_uploads = 20
   - upload_max_filesize = 2M
   - open_basedir = /path/directorio/publico/
   - upload_tmp_dir = /path/directorio/temporal/
   - allow_url_fopen = Off
   - allow_url_include = Off
6) Elimine POST y GET no esperados - sólo conserve los permitidos
7) Sanee las entradas del usuario o por URL que vaya a guardar en una base de datos o a mostrar en html
8) Valide las entradas del usuario o URL considerando su existencia, largo, tipo, formato, etc
9) Recuerde cuales son los ataques más usuales:
   - XSS
   - CSRF
   - SQL Injection
10) Utilice un token de validación en sus formularios y compárelo con su sesión
11) Valide que las peticiones son remitidas desde el mismo servidor/dominio
12) Nunca utilice un usuario de base de datos con todos los provilegios
13) Utilice consultas preparadas de ser posible, caso contrario siempre escape los datos
14) No guarde información sensible en cookies. Defina una fecha de expiración. Encripte sus datos.
